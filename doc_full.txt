  [6,0]: 一、课题来源、目的与要求：

课题来源：解决社会实际问题的课题。

目的：围绕个人及家庭成员的日常健康管理需求，设计并实现一个基于 Spring Boot 的个人健康管理与疾病预警系统。系统以健康档案、健康记录、风险评估、预警提示和成员详情分析为主线，解决健康数据分散、持续记录困难、异常指标不易被及时识别的问题。

要求：完成系统的分析、设计、实现与测试。系统应实现成员档案管理、健康记录管理、风险评估与预警、健康总览展示、成员详情分析及 AI 咨询接口等功能；采用前后端分离式接口组织方式，后端使用 Spring Boot、Spring MVC、Spring Data JPA，数据库采用 H2 文件数据库；前端使用 HTML、CSS、JavaScript 完成页面展示与交互；在毕业设计过程中完成需求分析、总体设计、详细设计、功能测试及设计说明书撰写。
  [7,0]: 二、主要设计内容：

本课题《基于SpringBoot的个人健康管理与疾病预警系统》主要设计内容如下：

1、健康总览模块
展示成员总数、健康记录总数、待处理预警数、高风险记录数、风险等级分布、最近记录和最近预警。
2、成员档案管理模块
实现家庭成员健康档案的新增、查询、修改和删除，维护姓名、关系、性别、年龄、血型、身高体重、慢病史、过敏史、用药史、运动习惯、健康目标、紧急联系人等信息。
3、健康记录管理模块
实现体重、腰围、血压、心率、血糖、体温、血氧、胆固醇、睡眠、运动、步数、饮水、压力、情绪、症状、用药等指标的录入、修改、删除和筛选展示。
4、风险评估与预警模块
依据源码中的阈值规则对 BMI、血压、血糖、血氧、血脂、睡眠、活动量、压力和情绪等指标进行评估，生成风险评分、风险等级和预警信息。
5、成员详情与 AI 咨询模块
聚合成员最新健康快照、近期记录、趋势指标、个性化建议与最近预警；在配置 DeepSeek API Key 的情况下，支持手动触发 AI 中文分析结果。
  [8,0]: 三、主要设计技术指标与参数：
1、开发语言与平台：Java 21，Spring Boot 3.5.11。
2、后端框架：Spring MVC、Spring Data JPA、Jakarta Validation。
3、数据库：H2 文件数据库，采用 MySQL 兼容模式，支持本地持久化演示。
4、前端实现：HTML + CSS + JavaScript，通过 fetch 调用 REST API。
5、系统结构：controller、service、repository、domain、dto、exception、config 分层组织。
6、核心技术要求：实现健康记录保存后自动进行风险评估、刷新预警信息，并支持成员详情聚合分析与可选 AI 咨询。
7、完成系统测试和设计说明书撰写，保证主要功能可运行、可演示、可复现。

  [2,0]: 一、课题的目的意义：
目的：基于Springboot开发一个面向个人与家庭成员使用的健康管理与疾病预警系统，实现健康档案维护、健康记录沉淀、异常指标识别、风险等级评估与预警提示，帮助用户形成持续记录与主动干预的健康管理方式。
意义：一方面，该系统能够把分散的个人健康信息集中管理，并通过规则化评估及时提示潜在风险；另一方面，项目采用 Spring Boot 分层架构、REST API、JPA 持久化和前端交互页面，具有较好的工程训练价值，适合作为软件工程专业毕业设计课题。
  [3,0]: 二、资料调研分析：
健康管理类系统通常包括档案管理、指标记录、趋势查看和异常预警等功能。结合本项目源码可见，系统采用家庭成员档案与健康记录关联的方式进行数据组织，围绕风险评估服务对多类生理指标进行阈值判断，再由预警模块输出提示信息，最终在成员详情页中整合近期趋势、建议和 AI 咨询入口。
从工程实现角度看，该项目没有采用复杂前端框架，而是使用 HTML、CSS 和原生 JavaScript 与 Spring Boot 后端接口配合，实现页面展示、弹窗表单、筛选和前端分页；后端以 controller、service、repository、domain、dto 等层次组织代码，适合进行需求分析、总体设计和详细设计说明。
  [4,0]: 三、设计方案的可行性分析和预期目标：
设计方案：系统以后端 Spring Boot 应用为核心，提供成员档案、健康记录、预警中心、健康总览和成员详情等 REST 接口；数据层采用 Spring Data JPA 访问 H2 文件数据库；前端使用 HTML、CSS、JavaScript 构建页面并调用接口；在服务器配置有效 DeepSeek API Key 时，可通过独立服务发起 AI 咨询请求。
可行性分析：
1、技术可行性：项目已具备完整源码，后端依赖清晰，使用 Spring Boot 3.5.11、Java 21、JPA 和 H2 数据库，页面交互由原生 JavaScript 实现，部署与演示门槛较低。
2、经济可行性：系统可在普通个人电脑上运行，无需额外商业数据库和前端框架投入，适合教学与答辩演示场景。
3、操作可行性：系统页面按“总览—档案—记录—预警—详情”组织，交互流程明确，用户可通过表单完成数据录入与查看。
预期目标：完成健康总览、成员档案管理、健康记录管理、风险评估与预警、成员详情分析以及可选 AI 咨询功能，实现主要业务流程联通并完成测试与文档撰写。
